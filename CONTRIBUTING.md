# Contributing Guidelines

Thank you for contributing to this project!  
All team members already have edit access and can directly work on the repository.

## How We Contribute

### 1. Create or Update a Branch
Before making changes:
- Create a new branch with a clear name.  
  Example: `phase1-update`, `fix-ui-bug`, `add-sos-api`.

### 2. Make Your Changes
- Write clean and readable code.
- Add comments where needed.
- Do not break existing features.
- Test your changes before committing.

### 3. Commit Guidelines
- Use meaningful commit messages.
  Example:  
  - `Added deepfake detection logic`  
  - `Improved pothole detection accuracy`  
  - `Fixed SOS location update bug`

### 4. Push and Create Pull Request (PR)
- Push your branch to GitHub.
- Open a Pull Request.
- Add a short explanation of what you changed.
- Another team member will review and approve.

### 5. Issue Reporting
If you find a bug or want a new feature:
1. Go to the **Issues** tab.  
2. Click **New Issue**.  
3. Describe the problem clearly.  

## Code Style
- Keep code modular and simple.
- Remove unused code and imports.
- Follow consistent formatting.

Thank you for helping improve this project!
